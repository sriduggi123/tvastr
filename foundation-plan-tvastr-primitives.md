**Frameworks Found**:
- C4 Model: useful for showing Tvastr as system/context/container/component relationships across CLI, MCP server, web server, artifact, and agent clients.
- arc42: useful as a comprehensive architecture documentation template covering context, constraints, building blocks, runtime behavior, deployment, and cross-cutting concerns.
- Diataxis: useful for splitting eventual external docs into tutorials, how-to guides, reference, and explanation, but less suited as the internal primitive foundation structure.
- OpenAPI + JSON Schema style contracts: useful for documenting web server APIs and the machine-readable mockup artifact as formal contracts.
- MCP concepts: useful for documenting Tvastr's MCP server in terms of tools, resources, prompts, roots, and local file access boundaries.
- DDD / domain modeling: useful for defining core entities such as Workspace, Project, Mockup, Screen, Component, Requirement, Annotation, Decision, Version, and Export.
- ADRs: useful later for individual architecture decisions, but not as the primary primitive foundation format.
